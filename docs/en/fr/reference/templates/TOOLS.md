# TOOLS.md - Notes Locales

Les compétences définissent _comment_ les outils fonctionnent. Ce fichier est pour _vos_ spécificités — les choses qui sont uniques à votre configuration.

## Ce qui va ici

Des choses comme :

- Noms et emplacements des caméras
- Hôtes SSH et alias
- Voix préférées pour TTS
- Noms des haut-parleurs/pièces
- Surnoms des appareils
- Tout ce qui est spécifique à l'environnement

## Exemples

```markdown
### Cameras

- living-room → Main area, 180° wide angle
- front-door → Entrance, motion-triggered

### SSH

- home-server → 192.168.1.100, user: admin

### TTS

- Preferred voice: "Nova" (warm, slightly British)
- Default speaker: Kitchen HomePod
```

## Pourquoi séparer ?

Les compétences sont partagées. Votre configuration est la vôtre. Les garder séparées signifie que vous pouvez mettre à jour les compétences sans perdre vos notes, et partager les compétences sans révéler votre infrastructure.
